分割代入(js)

分割代入とは、配列内の変数に対して、値を代入することを言います。
例えば、以下のコードを見てください。 const [x, y] = [10, 25]; console.log(x); // 10 console.log(y); // 25 これは、配列内で宣言された「x」「y」に対して、「[10, 25]」という配列の要素を分割して代入しています。
これを用いれば以下のようなことも可能です。 const entire_name = "hoge.txt"; const [name, ext] = entire_name.split("."); console.log(name); // 「hoge」 console.log(ext); // 「txt」 拡張子を含むファイル名を純粋なファイル名と拡張子に分割して代入しています。
これは、配列だけでなく、オブジェクトに対しても有効です。 const file = new Object(); const entire_name = "hoge.txt"; [file.name, file.ext] = entire_name.split("."); console.log(file); /* ↓ コンソール ↓ Object { name: 'hoge', ext: 'txt' } */
スプレット構文については、昨日か一昨日かに説明しましたね♪
おそらく、配列を展開する構文って感じで説明しましたが、分割代入の際には余りを取得する際に使用されることが多いです。 const en = "abcde".split(""); const [a, b, ...rest] = en; console.log(a); console.log(b); console.log(rest); /* ↓ コンソール ↓ a b Array(3) [ 'c', 'd', 'e' ] */ スプレット構文は説明した通り、余りを取得するため、分割代入の最後の変数であるべきです。

例えば、以下のコードはエラーとなります。 /* エラー */ const en = "abcde".split(""); const [...rest, a, b] = en; console.log(a); console.log(b); console.log(rest); /* ↓ コンソール ↓ Uncaught SyntaxError: rest element may not have a trailing comma */ 「...rest」が余りを全部取得しているので、それに続く「a」「b」に割り当てる値はありません。
