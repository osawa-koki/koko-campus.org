jsでの関数内関数

みなさん、関数内関数って知っていますか???
あまり有名ではないのですかね、、、

字面通り、関数内関数とは関数内で生成された関数を言います。 function out_fx(x, y) { function in_fx(p) { if (p.length < 5) { //処理1 //処理2 //処理3 in_fx(q); } } //処理1 //処理2 //処理3 in_fx(z); } ここでは、in_fxという関数がout_fxという関数内で生成されています。
そのため、in_fxが関数内関数となります。

なぜ関数内関数を使用するのでしょうか???

僕の解答は以下の2つです。
  • 再帰関数の実行
  • 変数汚染の防止
1つ目、再帰関数の実行に関しては当然に関数の名前がないと自分自身を呼び出せないため、関数に名前を付けています。
ではなぜ、関数内で生成しているのでしょうか???

この解答が関数内関数の2つ目の目的です。
このin_fxという関数はout_fxという関数の内部のみで再帰的に実行されるため、外部で生成する必要がないためです。
別に関数外で宣言してもok!ですが、そうするとin_fxという関数はもう生成できなくなります。
可読性の観点からこれは避けるべきでしょう。
