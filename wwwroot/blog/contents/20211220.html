関数式関数(js)

関数の定義の方法って知っていますか???
おそらく、皆さんが良く用いる(テキスト等に載っている)方法は関数を宣言する方法だと思いますが、javascriptではそれ以外にも関数を定義する方法があります。

javascriptで関数を定義する方法は2通りあります。
  1. 関数宣言
  2. 関数式

おそらく、javascriptで関数を作成するには以下のコードを記述する方法が主流だと思います。 function add(x, y) { return x + y; } console.log(add(10, 15)); //「25」と出力 最も一般的な方法で、関数を宣言する方法です。
しかしながら、この方法には以下のデメリットがあります。
  • 変数(関数)の巻き上げが発生
  • 関数の上書きが可能

変数(関数)の巻き上げが発生することはデメリットではなく、メリットとして扱うべきかもしれません、、、
この方法で関数を宣言した場合、関数を宣言するよりも前に関数を実行してもok!です。
したがって、以下のように記述しても動作します。 console.log(add(10, 15)); //「25」と出力 function add(x, y) { return x + y; } 問題は関数の上書きが可能なことです。
通常のプログラミングでは関数を上書きすることなんてありません。
少なくとも僕は10年弱プログラミングしていますが、変数を上書きすることはあっても関数を上書きしたことはありません。

コード量が膨大になった際に間違って関数を上書きしてしまわないように関数の上書きが発生した際にエラーとして捉えたいですね♪

ということで、先ほどの関数宣言関数を上書き不可なconstを用いた関数式で記述してみましょう♪ const add = function(x, y) { return x + y; } console.log(add(10, 15)); //「25」と出力 functionによる関数の宣言やvar関数式による関数の作成と異なり、constでは変数の巻き上げが行われないため以下のコードはエラーとなります。 console.log(add(10, 15)); //エラー(Uncaught ReferenceError: Cannot access 'add' before initialization) const add = function(x, y) { return x + y; } また、関数を上書きしようとするとエラーとなります。 const add = function(x, y) { return x + y; } const add = function(x, y, z) { return x + y + z; } console.log(add(10, 15)); 上のコードを実行すると「Uncaught SyntaxError: Identifier 'add' has already been declared」とエラーとなります。

絶対と言っていいほど、関数を上書きする機会はありませんので、関数はconstを用いた関数式で作成する癖を付けましょう♪

では、さらに進化バージョンである「const関数式」 + 「アロー関数」で関数を作成しましょう♪
