アロー関数(js)

皆さん、javascriptで関数ってどう定義していますか???
おそらく以下のどちらかだと思います。
  • function
  • =>
それぞれ、「通常関数」「アロー関数」と呼びますが、この違いって知っていますか???
アロー関数の方が新しくて簡単に書けるからアロー関数を使えばok!ってわけにはいきません。
両者を適切に使い分ける必要があります。
理解するべき大きな違いは「this」が何を指すかです。
例えば以下のコードを見てください。 <input id="name" type="text" /> "use strict"; document.getElementById("name").addEventListener("change", function() { console.log(this); }); id属性に「name」が設定されたテキストボックスに変更時に実行されるイベントリスナーを設定しています。
イベントリスナーで発火される関数内でthisを使用していますが、この「this」はid属性に「name」が設定されたテキストボックス(ドキュメントオブジェクト)を指します。

通常関数内での「this」はそのオブジェクトメソッドが含まれる直近のオブジェクトを指します。
上の例では、addEventListenerメソッドが含まれる直近のオブジェクトであるid属性に「name」が設定されたテキストボックス(ドキュメントオブジェクト)を指します。

これをアロー関数を用いて関数を定義した以下のコードに変更してみましょう♪ <input id="name" type="text" /> "use strict"; document.getElementById("name").addEventListener("change", () => { console.log(this); }); アロー関数において、「this」は関数が定義されているスコープを指します。
そのため、上のコードでの「this」はwindowオブジェクトを指します。
